FlowAI / How It Works
System Architecture
Four-stage AI pipeline for real-time traffic intelligence

FlowAI combines real-time IoT sensor data, graph theory, and machine learning to model, predict, and smooth urban traffic flow. The system operates as a four-stage pipeline — from raw sensor ingestion to actionable AI recommendations delivered in under 2 seconds.

01
📡
Real-Time Data Collection
IoT sensors embedded at every major intersection stream vehicle counts, average speeds, and queue lengths every 30 seconds. GPS traces from connected vehicles are aggregated and anonymized. The raw feed is cleaned, normalized, and written to a distributed time-series database with sub-second latency.
IoT Sensors GPS Traces Time-Series DB 30s Updates
02
🕸️
Graph Modeling
The city road network is modeled as a weighted directed graph G(V, E). Each node V represents an intersection; each edge E carries a dynamic weight derived from current traffic density, road capacity, speed limits, and historical congestion patterns. Edge weights are recomputed every 30 seconds from live sensor feeds.
Directed Graph Dynamic Weights Road Capacity Live Updates
03
⚡
Dijkstra's Shortest Path Algorithm
When a user requests a route, Dijkstra's algorithm runs on the live-weighted graph to find the path that minimizes total weighted travel cost — not just geographic distance, but actual time factoring in current congestion. The algorithm explores all reachable nodes using a priority queue (min-heap), guaranteeing the globally optimal result.
Dijkstra's Algorithm Min-Heap PQ O((V+E) log V) Optimal Path
04
🤖
AI Flow Smoothening
A trained gradient-boosted model analyzes historical patterns, current density, time-of-day, and day-of-week to predict upcoming bottlenecks 15–20 minutes ahead. The system recommends adaptive traffic signal timing adjustments, redistributes load to alternate corridors, and issues congestion warnings — all before gridlock forms.
Gradient Boosting Predictive Alerts Signal Timing Load Balancing
Dijkstra's Algorithm — Pseudocode
function dijkstra(graph, src, dst): dist[src] = 0 dist[v] = ∞ for all other v pq = MinHeap() pq.push(0, src) while pq not empty: (d, u) = pq.pop() if u == dst: break for edge (u→v, weight) in graph[u]: nd = d + weight if nd < dist[v]: dist[v] = nd prev[v] = u pq.push(nd, v) // Reconstruct path path = [] cur = dst while cur: path.prepend(cur); cur = prev[cur] return path, dist[dst]
Edge Weight Formula
Each road segment's weight is dynamically computed from three factors: base distance, real-time traffic load, and a time-of-day multiplier learned from historical data.
// Dynamic edge weight weight(e) = distance(e) × (1 + congestion_factor(e)) × time_multiplier(hour) // Congestion factor (0.0–1.2) congestion_factor = (density / capacity) ^ 1.4 // Time multiplier from ML model time_multiplier = if rush_hour: 1.4 – 1.9 if off_peak: 0.6 – 0.9 else: 1.0
